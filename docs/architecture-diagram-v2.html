LAYER 1 User Interfaces Web, SMS, and IVR access channels
▼
WEB
Web Application
React 18 + Vite + TailwindCSS
Single-page application serving all user roles. Built with React 18, bundled with Vite 5, styled with TailwindCSS.
  • TrackingDashboard — Passenger bus search and ETA
  • EmergencyDashboard — Safety info and helplines
  • ServiceReporting — Feedback and complaints
  • OperatorDashboard — Trip management
  • Login / OTP — Operator authentication
  • RouteDetails — Detailed route view with stops
SMS
SMS Channel
Webhook-based SMS Parser
Parses incoming SMS queries in natural format for feature-phone accessibility.
  • Format: BUS 42B Adyar
  • Processes via /api/v1/sms/webhook
  • Response delivered via AWS SNS
  • Rate limited: 10 requests/hour/phone
IVR
IVR Channel
Voice-Based Interface (Framework Ready)
Voice-based transit query system for non-smartphone users. Architecture designed, implementation planned.
  • Rate limited: 5 calls/hour/phone
  • Multilingual support planned (Tamil, English, Hindi)
  • Amazon Connect integration ready
LAYER 2 CDN and Distribution Global edge caching and static hosting
▼
CF
Amazon CloudFront
Global CDN and Edge Cache
Content Delivery Network with 400+ edge locations worldwide. Caches static assets and proxies API requests.
  • HTTPS termination with ACM certificates
  • Origin: S3 bucket (static) + API Gateway (dynamic)
  • Gzip/Brotli compression enabled
  • Custom error pages for 4xx/5xx
S3
S3 Static Website Hosting
Frontend Asset Origin
Hosts the compiled React SPA build artifacts. Bucket configured with website hosting and OAI for CloudFront.
  • Bucket: transit-india-frontend-*
  • Versioning enabled for rollbacks
  • OAI (Origin Access Identity) configured
  • Build output from vite build
LAYER 3 API Layer Gateway, rate limiting, and security
▼
API
Amazon API Gateway
REST API — CORS Enabled
Managed REST API gateway that routes all HTTP requests to the Lambda backend. Handles request validation and throttling.
  • Stage: Prod
  • CORS: All origins (development)
  • Binary media type passthrough
  • CloudWatch access logging enabled
RL
Rate Limiter
Redis-Backed Throttling
Protects endpoints from abuse using sliding-window rate limiting with Redis counters.
SMS 10/hr/phone
IVR 5/hr/phone
Safety 3/day/phone
WAF
AWS WAF
Web Application Firewall
Protects API Gateway from common web exploits: SQL injection, XSS, and DDoS mitigation.
  • AWS Managed Rules enabled
  • IP reputation filtering
  • Geographic restrictions (India focus)
  • Request size limits enforced
LAYER 4 Compute (Serverless) Lambda function and FastAPI application
▼
λ
AWS Lambda
transit-india-api
Serverless compute function running the entire backend application. Zero infrastructure management.
  • Runtime: Python 3.9
  • Memory: 512 MB
  • Timeout: 30 seconds
  • Handler: Mangum (ASGI adapter)
  • Deployed via SAM / CloudFormation
  • VPC-attached (private subnets)
API
FastAPI Application
ASGI Web Framework
High-performance async Python framework with automatic OpenAPI docs, Pydantic validation, and dependency injection.
  • SQLAlchemy ORM with async sessions
  • Pydantic v2 request/response models
  • CORS middleware configured
  • Structured JSON logging
  • Health check endpoint

ConfidenceCalculator
Rule-Based Scoring Engine
Computes prediction confidence (0-100%) using data freshness, GPS availability, historical accuracy, and time-of-day factors.
  • Source: confidence.py

  • Returns confidence + reasoning text

LAYER 6 Data Layer PostgreSQL, Redis, and S3 storage
▼
RDS
RDS PostgreSQL 15
Primary Relational Database
Managed PostgreSQL instance in private subnets. Handles all transactional data with multi-AZ deployment.
  • Engine: PostgreSQL 15
  • Instance: db.t3.micro (free tier)
  • Encrypted at rest (KMS)
  • Automated backups: 7-day retention
EC
ElastiCache Redis
In-Memory Cache and Store
Sub-millisecond cache for trip status, rate limiting counters, and session management.
  • Trip status cache (TTL: 60s)
  • Rate limiting counters (sliding window)
  • Session cache for auth tokens
  • Instance: cache.t3.micro
S3
S3 Buckets
Object Storage
Three dedicated S3 buckets for different data classes.
  • transit-india-frontend-* — Static assets
  • transit-india-audit-* — 90-day audit logs
  • transit-india-ml-* — Model artifacts
  • Lifecycle policies for cost optimization

LAYER 8 External Services Third-party integrations and data sources
▼
SNS
Amazon SNS
OTP SMS Delivery
Transactional SMS delivery for operator OTP authentication. Configured for India (+91) with sender ID.
  • Type: Transactional SMS
  • Region: ap-south-1
  • OTP format: 6-digit, 5-min expiry
  • Fallback: Debug OTP in response
ETM
ETM Systems
Electronic Ticketing Machines
Bus-mounted ticketing devices that generate login and ticket events used for trip data ingestion.
  • Bus login/logout events
  • Ticket issuance data
  • Real-time data pipeline planned
GPS
GPS Data Sources
Vehicle Location Tracking
Optional GPS feeds from bus-mounted trackers. Currently sparse; system designed to work with or without GPS data.
  • NMEA data format support
  • Fallback to schedule-based ETA
  • Confidence adjusts with GPS availability
LAYER 9 Monitoring and Security Observability, audit, and access control
▼
CW
CloudWatch
Logs + Metrics
Centralized logging and monitoring for all Lambda invocations, API Gateway access, and custom application metrics.
  • Log groups per Lambda function
  • Custom metrics: latency, errors
  • Alarms for error rate thresholds
AL
S3 Audit Logs
90-Day Lifecycle
Structured audit trail for all safety escalations, operator actions, and administrative operations.
  • JSON-structured log entries
  • S3 lifecycle: 90-day expiry
  • Queryable via Athena (planned)
IAM
IAM Roles and Policies
Least-Privilege Access
Fine-grained IAM roles for Lambda execution, S3 access, RDS connectivity, and SNS publishing.
  • Lambda execution role
  • S3 read/write policies
  • RDS connect permissions
  • SNS publish permissions
BK
AWS Backup
Automated Backups
Automated backup plans for RDS and critical data with cross-region copy for disaster recovery.
  • RDS: 7-day automated snapshots
  • S3: Versioning enabled
  • Recovery point objective: 1 hour
